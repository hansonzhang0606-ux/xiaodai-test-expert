# 安全测试用例模板

> 用于数据安全、权限控制、输入校验、敏感信息保护类测试用例

---

```markdown

**功能路径**：{模块}-{安全场景}

**前置条件**：
- {权限配置}

1、{结果1：权限校验生效}
```

## 安全测试用例设计要点

| 要点 | 说明 |
|------|------|
| **数据加密** | 每个敏感字段至少覆盖：加密存储、传输加密、解密验证 |
| **权限控制** | 每个权限点至少覆盖：有权限通过、无权限拒绝 |
| **输入校验** | 每个输入点至少覆盖：合法输入、非法输入、注入攻击 |
| **日志脱敏** | 每个敏感数据至少覆盖：日志脱敏、完整日志不含敏感信息 |
| **接口安全** | 每个敏感接口至少覆盖：Token校验、签名校验、防重放 |

---

## 常见安全测试场景

| 场景类型 | 测试要点 |
|---------|---------|
| **敏感数据存储** | 加密存储、加密格式正确、解密验证 |
| **敏感数据传输** | HTTPS传输、证书校验 |
| **权限控制** | 有权限通过、无权限拒绝、越权拒绝 |
| **输入校验** | SQL注入、XSS攻击、特殊字符处理 |
| **日志安全** | 敏感信息脱敏、日志不含明文密码/身份证 |

---

## 安全风险等级定义

| 风险等级 | 说明 | 用例级别 |
|---------|------|----------|
| **高** | 涉及用户隐私数据、财务数据、可能导致数据泄露 | P0 |
| **中** | 涉及权限控制、可能导致越权操作 | P1 |
| **低** | 涉及输入校验、日志规范 | P2 |
