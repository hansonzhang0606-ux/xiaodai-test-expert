# 状态流转测试模式

> 本文档定义状态流转类需求的测试场景，适用于涉及状态机、工作流的需求

---

## 📋 状态流转测试场景

### 1. 状态展示类

| 测试场景 | 测试要点 |
|---------|---------|
| 各状态前端展示正确 | 验证每种状态前端展示文案/样式正确 |
| 状态图标/颜色正确 | 验证状态对应的图标/颜色正确 |
| 状态变更提示正确 | 验证状态变更时提示信息正确 |

### 2. 状态流转类

| 测试场景 | 测试要点 |
|---------|---------|
| 正常流转路径验证 | 验证正常状态流转路径可触发 |
| 流转触发条件验证 | 验证每个流转触发条件正确 |
| 流转顺序验证 | 验证状态必须按顺序流转（不可跳跃） |
| 流转后数据更新 | 验证流转后状态字段正确更新 |

### 3. 状态边界类

| 测试场景 | 测试要点 |
|---------|---------|
| 状态有效期边界 | 验证状态有效期边界（如14天23:59:59） |
| 状态过期翻转 | 验证状态过期后正确翻转为过期状态 |
| 状态翻转时机 | 验证状态翻转时机正确（定时任务/API） |

### 4. 非法流转类

| 测试场景 | 测试要点 |
|---------|---------|
| 非法流转拦截 | 验证非法状态流转被正确拦截 |
| 逆向流转验证 | 验证不可逆转的状态无法逆向流转 |
| 跨状态流转验证 | 验证不可跨状态的流转被拦截 |

### 5. 状态并发类

| 测试场景 | 测试要点 |
|---------|---------|
| 状态并发变更 | 验证多用户同时变更同一状态时一致性 |
| 状态锁验证 | 验证状态变更时正确加锁 |
| 状态变更冲突处理 | 验证状态变更冲突时正确处理 |

### 6. 状态持久化类

| 测试场景 | 测试要点 |
|---------|---------|
| 状态变更时间记录 | 验证状态变更时间正确记录 |
| 状态变更人记录 | 验证状态变更操作人正确记录 |
| 状态变更历史查询 | 验证状态变更历史可查询 |
| 状态数据持久化 | 验证状态数据正确持久化到数据库 |

### 7. 状态恢复类

| 测试场景 | 测试要点 |
|---------|---------|
| 异常状态恢复 | 验证异常状态可恢复到正常状态 |
| 状态回退验证 | 验证可回退状态正确回退 |
| 状态重置验证 | 验证状态重置功能正确 |

---

## 🔍 状态机分析检查清单

当需求涉及状态流转时，AI 应检查：

| 检查项 | 检查内容 |
|--------|---------|
| 状态定义完整？ | 所有状态是否已定义 |
| 状态流转规则完整？ | 每个状态的流转条件是否已说明 |
| 状态流转图是否存在？ | 是否有状态流转图可参考 |
| 状态边界是否定义？ | 状态有效期/过期规则是否已定义 |
| 异常状态处理是否说明？ | 异常状态如何处理是否已说明 |

---

## 📝 状态流转图解析要点

当文档中包含状态流转图时，AI 应提取：

| 提取项 | 说明 |
|--------|------|
| 状态列表 | 所有状态名称和编码 |
| 流转路径 | 状态之间的流转关系 |
| 流转条件 | 每个流转的触发条件 |
| 流转动作 | 流转时执行的动作 |
| 边界状态 | 有效期/过期等边界状态 |
