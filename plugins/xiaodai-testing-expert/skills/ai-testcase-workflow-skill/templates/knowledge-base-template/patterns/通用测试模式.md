# 通用测试模式

> 本文档定义适用于所有需求的通用测试场景，AI 应在生成测试点时自动补充这些场景

---

## 📋 通用测试场景清单

### 1. 权限控制类

| 测试场景 | 测试要点 |
|---------|---------|
| 无权限访问拒绝 | 验证无权限用户访问功能时正确拒绝 |
| 权限边界验证 | 验证权限边界用户（刚好有/刚好无）处理正确 |
| 权限继承验证 | 验证子权限与父权限关系正确 |
| 越权操作拦截 | 验证横向越权、纵向越权被正确拦截 |

### 2. 数据边界类

| 测试场景 | 测试要点 |
|---------|---------|
| 空值处理 | 验证空字符串、空数组、null 值处理正确 |
| 最大值边界 | 验证字段最大长度、最大数量处理正确 |
| 最小值边界 | 验证字段最小长度、最小数量处理正确 |
| 超界值处理 | 验证超出范围值正确提示或拒绝 |

### 3. 输入校验类

| 测试场景 | 测试要点 |
|---------|---------|
| 必填字段校验 | 验证必填字段为空时正确提示 |
| 格式校验 | 验证邮箱、手机号、身份证等格式校验正确 |
| 类型校验 | 验证数值型、日期型字段类型校验正确 |
| 特殊字符处理 | 验证特殊字符输入处理正确 |

### 4. 并发场景类

| 测试场景 | 测试要点 |
|---------|---------|
| 重复提交防护 | 验证重复点击按钮不重复发起请求 |
| 并发数据一致性 | 验证多用户同时操作同一数据时一致性 |
| 分布式锁验证 | 验证分布式锁正确加锁、释放锁 |

### 5. 异常处理类

| 测试场景 | 测试要点 |
|---------|---------|
| 网络超时处理 | 验证接口超时时正确提示或重试 |
| 服务异常处理 | 验证依赖服务异常时降级处理正确 |
| 数据异常处理 | 验证异常数据（损坏、缺失）处理正确 |
| 异常恢复验证 | 验证异常恢复后系统状态正确 |

### 6. 日志与监控类

| 测试场景 | 测试要点 |
|---------|---------|
| 关键操作日志 | 验证关键操作正确记录日志 |
| 异常告警触发 | 验证异常场景正确触发告警 |
| 日志脱敏验证 | 验证敏感信息在日志中正确脱敏 |
| TraceId 链路追踪 | 验证全链路 TraceId 正确传递 |

### 7. 回滚与恢复类

| 测试场景 | 测试要点 |
|---------|---------|
| 操作回滚验证 | 验证操作失败时数据正确回滚 |
| 数据恢复验证 | 验证删除/修改操作可恢复 |
| 状态恢复验证 | 验证异常后状态正确恢复 |

## 🔍 AI 补充检查清单

生成测试点后，AI 应检查是否遗漏以下场景：

| 检查项 | 需求特征 | 补充场景 |
|--------|---------|---------|
| 是否有权限相关描述？ | 提到"角色"、"权限" | 权限控制类场景 |
| 是否有数值范围定义？ | 提到"最大"、"最小"、"范围" | 数据边界类场景 |
| 是否有输入字段定义？ | 提到"输入"、"填写"、"参数" | 输入校验类场景 |
| 是否有并发场景描述？ | 提到"多用户"、"同时" | 并发场景类场景 |
| 是否有第三方依赖？ | 提到"接口"、"第三方"、"外部" | 异常处理类场景 |
| 是否有日志要求？ | 提到"日志"、"埋点"、"监控" | 日志监控类场景 |

---

## 📝 使用说明

1. **AI 生成测试点时**：对照清单检查遗漏场景
2. **发现遗漏时**：自动补充对应测试场景
3. **记录补充来源**：在 JSON 中标注 `source: "knowledge-base"`
