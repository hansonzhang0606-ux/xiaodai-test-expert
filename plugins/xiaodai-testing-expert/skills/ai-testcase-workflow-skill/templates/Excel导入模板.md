# Excel 导入模板说明

> 本文档说明测试用例 Excel 的导入格式要求

---

## Excel 文件结构

| 行号 | 内容 | 说明 |
|------|------|------|
| **第1行** | `用例管理 # dmp_testcase` | 标题行（固定） |
| **第2行** | 字段录入要求说明 | 说明行（固定） |
| **第3行** | 列名行 | team, caseGroup, name, ... |
| **第4行** | 字段说明行 | *项目组, *功能路径, ... |
| **第5行起** | 测试用例数据 | 真正的用例内容 |

---

|------|------|----------|------|------|

## 字段填写规范

### caseGroup（功能路径）

格式：`年份-月份-日期-需求名称`

示例：
- `2026-202603-20260326-需求名称`

### caseType（用例类型）

可选值：
- 功能测试
- 集成测试
- 安全漏洞
- 性能测试

### source（来源）

⚠️ **只能填写以下三个值**：
- 需求文档
- 产品用例
- 补丁用例

### caseLevel（用例级别）

| 级别 | 适用场景 |
|------|---------|
| P0 | 核心流程、关键功能 |
| P1 | 一般功能（默认） |
| P2 | 边缘场景、次要功能 |
| P3 | 优化建议类 |

### separator（分隔符）

⚠️ **分隔符为空，不填写**

---

## 步骤格式规范

**必须使用「1、」「2、」格式**：

```
✅ 正确格式：
1、完成签署流程
2、观察提示内容

❌ 错误格式：
1. 完成签署流程    ← 禁止使用点+空格
```

**换行分隔**：

步骤和预期结果使用 `\n` 换行分隔。

---

## 注意事项

1. **步骤数量 ≥ 2**：每个用例至少2个步骤
2. **步骤与预期对应**：步骤数 = 预期结果数
3. **数据实例化**：使用具体数据，禁止"某值"占位符
4. **必填字段**：team、caseGroup、name、product、modulePath、caseType、manager
