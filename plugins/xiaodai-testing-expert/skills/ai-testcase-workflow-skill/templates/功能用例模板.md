# 功能测试用例模板

> 用于单模块业务功能、规则校验、界面交互、字段验证类测试用例

---

## 用例结构

```markdown
### {用例编号}：{用例名称}

**用例类型**：功能测试
**用例级别**：P1（默认）/ P0（核心流程）
**功能路径**：{模块}-{子模块}-{功能}

**前置条件**：
- {前置条件1}
- {前置条件2}
- {前置条件3}

**步骤**：
1、{步骤1：操作动作}
2、{步骤2：查看/验证}
3、{步骤3：操作动作}

**预期结果**：
1、{结果1：页面状态}
2、{结果2：字段值}
3、{结果3：提示信息}

**来源**：需求文档
**关联需求**：{需求编号}
```

---

## 功能测试用例设计要点

| 要点 | 说明 |
|------|------|
| **字段校验** | 每个新增字段至少覆盖：正常值、异常值、边界值、空值 |
| **规则校验** | 每条业务规则至少覆盖：符合规则、不符合规则 |
| **按钮操作** | 每个按钮至少覆盖：正常点击、无权限点击、重复点击 |
| **页面展示** | 每个页面元素至少覆盖：正常展示、异常数据展示 |
| **状态展示** | 每个状态值至少覆盖：展示正确、文案正确 |

---

## 常见功能测试场景

| 场景类型 | 测试要点 |
|---------|---------|
| **字段新增** | 展示正确、取值正确、边界验证 |
| **字段修改** | 修改成功、校验规则生效、历史数据兼容 |
| **字段删除** | 删除成功、关联数据处理 |
| **按钮新增** | 点击响应、权限控制、重复点击处理 |
| **页面新增** | 元素展示、交互正确、数据加载 |
| **流程新增** | 流程完整、分支覆盖、异常处理 |
