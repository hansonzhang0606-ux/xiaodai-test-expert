# Confluence 页面提取与整理 Prompt

> 此 Prompt 用于 AI 将用户提供的 **Confluence 在线页面 URL** 提取、转换为结构化 Markdown，作为步骤①「文档整理」的**轻量替代入口**。
>
> 与本地目录整理（document_consolidate.md）**并行存在**：
> - 有本地文件目录 → 走 `document_consolidate.md`
> - 只有 Confluence 在线页面 → 走本 Prompt
>
> 提取完成后，产出的 `{页面标题}_整理版_v1.0.md` 与本地整理版 MD **完全等价**，可直接进入步骤②「需求评审」。

---

## ⛔ 禁止 AI 擅自做决定（强制按流程规范执行）

| 禁止项 | 说明 | 正确做法 |
|--------|------|---------|
| ⛔ 禁止假设已读取 | 不假设前置步骤已完成，必须实际执行每步读取操作 | 每个步骤按规范重新读取 |
| ⛔ 禁止跳过预检 | Confluence 环境预检（环境变量/连通性/页面权限）必须执行 | 按 confluence-to-md skill 要求执行 |
| ⛔ 禁止自动触发评审 | 提取完成后不自动执行需求评审 | 必须等待用户明确指令（如"评审这个需求"） |
| ⛔ 禁止修改 Confluence | 本流程只读取页面，绝不修改/删除/创建页面 | 仅调用读取类工具 |
| ⛔ 禁止暴露内部地址 | 提示用户配置时，不得将真实域名/IP 填入示例 | 使用通用占位符 |

---

## ⚠️ 执行前必须创建 Todo 清单 ⭐

**Confluence 提取子 Todo 清单（6 项强制步骤）**：

| 序号 | 步骤 | 强制 | 输出/验证 |
|------|------|------|----------|
| 1 | 提取 Confluence URL / pageId | ⭐ 强制 | 从用户消息中识别 |
| 2 | 确认输出目录 | ⭐ 强制 | 询问用户保存位置，或使用当前项目目录 |
| 3 | 调用 confluence-to-md 能力提取页面 | ⭐ 强制 | 生成原始 MD + images/ |
| 4 | 转换为 `{页面标题}_整理版_v1.0.md` | ⭐ 强制 | 按本文档固定结构组织 |
| 5 | 创建/更新 `source/.整理索引.yaml` | ⭐ 强制 | 标记来源为 Confluence |
| 6 | 输出核对提示，等待用户校验 | ⭐ 强制 | 不自动触发评审 |

## 触发方式

用户消息符合以下任一模式时，进入本流程：

| 触发模式 | 示例 |
|---------|------|
| 发送 Confluence URL + 提取指令 | `https://xxx/pages/viewpage.action?pageId=12345 帮我提取这个文档内容` |
| 明确使用 Confluence 整理 | `整理这个 Confluence 页面：xxx` |
| 发送 URL + 后续指令 | `帮我提取这个文档内容`（当前消息或上一条消息包含 URL） |

> 如果用户只发了 URL 没有明确指令，**先询问意图**，不要直接提取。

## 核心原则：用户核对优先

```
用户：发送 Confluence URL + "帮我提取这个文档内容"
    │
    ↓
AI 提取页面 → 输出 {页面标题}_整理版_v1.0.md
    │
    ↓
⏸️ 暂停！等待用户校验 MD
    │
    ├── 用户确认正确 → AI 标记"已确认"，等待下一步指令
    │
    ├── 用户指出错误 → AI 根据描述修正 MD → 再次等待校验
    │
    └── 用户补充说明 → AI 将补充内容写入 MD → 再次等待校验
    │
    ↓
用户说"评审这个需求" → 进入步骤② 需求评审
```

> 提取完成后不自动触发评审！必须等待用户明确指令。

## 执行步骤

### 步骤 1：提取 URL / pageId ⭐

从用户消息中提取 Confluence 页面地址。支持格式：

- `https://your-confluence-server/pages/viewpage.action?pageId=123456`
- `https://your-confluence-server/display/SPACE/Page+Title`
- `123456`（纯 pageId，需结合环境变量 CONFLUENCE_URL）

**如果用户没有提供 URL**：

```
请提供需要提取的 Confluence 页面链接，或告知 pageId。
```

---

### 步骤 2：确认输出目录 ⭐

**优先使用当前项目目录**（如果会话中已缓存项目根目录）。

**如果未缓存项目目录**：

```
请告诉我把提取后的整理版 MD 保存到哪个目录？
例如：D:\项目文件\效贷\2026Q3\贷款审批优化
```

---

### 步骤 3：调用 Confluence 提取能力 ⭐

**方式 A（优先）：使用已安装的 confluence-to-md skill**

调用 Skill：`confluence-to-md`

执行：

```bash
python confluence_to_md.py "<页面URL或pageId>" -o ./output
```

> 脚本会自动执行三阶段预检（环境变量/连通性/页面权限）。按 skill 说明处理预检结果。

**方式 B（skill 不可用时）：使用 MCP confluence 连接器工具**

如果 WorkBuddy 中已配置并启用 confluence MCP 连接器，可直接调用：

- `mcp__confluence__confluence_get_page` / `mcp__confluence__confluence_get_page_by_url` 读取页面内容
- `mcp__confluence__confluence_get_page_attachments` / `mcp__confluence__confluence_get_page_body_chunk` 读取附件和正文

将获取到的内容转换为 Markdown 格式。

**⚠️ 安全约束**：
- 只读取，不修改 Confluence
- 不得暴露内部服务器地址
- 敏感内容页面需先询问用户确认

---

### 步骤 4：转换为整理版 MD ⭐

将提取到的原始 MD 按以下固定结构重组：

```markdown
# {页面标题} 整理版

> 整理日期：YYYY-MM-DD
> 版本：v1.0
> 来源：Confluence 页面
> 源 URL：{原始 Confluence URL}
> MD 版本历史：
> - v1.0（YYYY-MM-DD）：首次从 Confluence 提取
```

在输出目录创建 `source/.整理索引.yaml`：

```yaml

version: "v1.0"
需求名称: "{页面标题}"
整理日期: "YYYY-MM-DD"
MD版本: "v1.0"
MD文件: "{页面标题}_整理版_v1.0.md"
来源类型: "Confluence"
源URL: "{原始URL}"

源文件清单:
  - 文件名: "{页面标题}"
    类型: "Confluence页面"
    来源: "Confluence"
    URL: "{原始URL}"
    转换状态: "已转换"
    转换输出: "章节一"
    校验状态: "待确认"
```

```

📄 输出文件：{输出目录}\{页面标题}_整理版_v1.0.md

📊 整理索引：{输出目录}\source\.整理索引.yaml

⚠️ 请校验：
1. 打开整理版 MD，查看正文内容是否完整
2. 查看页面中的图片/附件是否已正确归档到 source/images/
3. 重点核对 AI 解析图片中的 `??` 标记不确定项
4. 确认无误后回复「确认整理完成」，或指出需要修正的地方

✅ 下一步：
- 说「评审这个需求」→ 进入步骤② 需求评审
- 说「继续」→ 我等待你的下一步指令

⛔ 我不会自动开始评审，等待你的确认指令。
```

## 与步骤②「需求评审」的衔接

当用户回复「确认整理完成」后，整理版 MD 已准备就绪。

**用户进入步骤②的方式**：

| 用户指令 | AI 行为 |
|---------|---------|
| `评审这个需求` | 读取当前 `{页面标题}_整理版_v1.0.md`，按 `requirement_review.md` 执行评审 |
| `评审这个目录` | 询问具体目录，进入 `document_consolidate.md` 或 `requirement_review.md` |
| `走完整流程` | 从步骤②开始继续执行（步骤①已完成） |

**需求评审前置检查**（requirement_review.md 0-3）看到 `*整理版*.md` 存在即可继续，不区分来源是本地目录还是 Confluence。

## 注意事项

1. **Confluence 页面权限**：如果页面 404/403，告知用户检查 URL 或联系管理员开通权限，**不要反复提示配置环境变量**。
2. **图片识别**：页面中的图片默认下载到 `source/images/`，如果未配置 ANTHROPIC_AUTH_TOKEN，图片 AI 识别会跳过，但仍正常导出文字内容。
3. **大页面处理**：如果页面内容很长，提取时间可能较长，提前告知用户。
4. **多页面处理**：如果用户要求处理多个 Confluence 页面，先列出页面列表，经用户确认后再逐个执行。
5. **敏感内容**：页面标题包含"机密/保密/内部/confidential/internal"等关键词时，暂停执行并询问用户确认。
