# Confluence 页面提取与整理 Prompt

> 此 Prompt 用于 AI 将用户提供的 **Confluence 在线页面 URL** 提取、转换为结构化 Markdown，作为步骤①「文档整理」的**轻量替代入口**。
>
> 与本地目录整理（document_consolidate.md）**并行存在**：
> - 有本地文件目录 → 走 `document_consolidate.md`
> - 只有 Confluence 在线页面 → 走本 Prompt
>
> 提取完成后，产出的 `{页面标题}_整理版_v1.0.md` 与本地整理版 MD **完全等价**，可直接进入步骤②「需求评审」。

---

## ⛔ 禁止 AI 擅自做决定（强制按流程规范执行）

| 禁止项 | 说明 | 正确做法 |
|--------|------|---------|
| ⛔ 禁止假设已读取 | 不假设前置步骤已完成，必须实际执行每步读取操作 | 每个步骤按规范重新读取 |
| ⛔ 禁止跳过预检 | Confluence MCP 连接器可用性检查必须执行 | 按本 Prompt 步骤 3 的预检流程执行 |
| ⛔ 禁止自动触发评审 | 提取完成后不自动执行需求评审 | 必须等待用户明确指令（如"评审这个需求"） |
| ⛔ 禁止修改 Confluence | 本流程只读取页面，绝不修改/删除/创建页面 | 仅调用读取类工具 |
| ⛔ 禁止暴露内部地址 | 提示用户配置时，不得将真实域名/IP 填入示例 | 使用通用占位符 |

## ⚠️ 执行前必须创建 Todo 清单 ⭐

**Confluence 提取子 Todo 清单（7 项强制步骤）**：

| 序号 | 步骤 | 强制 | 输出/验证 |
|------|------|------|----------|
| 0 | 确认用户故事信息并定位工作目录 | ⭐ 强制 | 向测试人员确认DMP用户故事编号和名称，创建/定位 `D:\效贷-产品需求\{用户故事编号}-{用户故事名称}` 目录 |
| 1 | 提取 Confluence URL / pageId | ⭐ 强制 | 从用户消息中识别 |
| 2 | 确认输出目录（使用用户故事工作目录） | ⭐ 强制 | 使用步骤0确认的用户故事目录 `{user_story_dir}` |
| 3 | 通过 MCP 连接器提取页面内容 | ⭐ 强制 | 生成原始 MD + images/ |
| 4 | 转换为 `{页面标题}_整理版_v1.0.md` | ⭐ 强制 | 按本文档固定结构组织 |
| 5 | 创建/更新 `source/.整理索引.yaml` | ⭐ 强制 | 标记来源为 Confluence |
| 6 | 输出核对提示，等待用户校验 | ⭐ 强制 | 不自动触发评审 |

## 触发方式

用户消息符合以下任一模式时，进入本流程：

| 触发模式 | 示例 |
|---------|------|
| 发送 Confluence URL + 提取指令 | `https://xxx/pages/viewpage.action?pageId=12345 帮我提取这个文档内容` |
| 明确使用 Confluence 整理 | `整理这个 Confluence 页面：xxx` |
| 发送 URL + 后续指令 | `帮我提取这个文档内容`（当前消息或上一条消息包含 URL） |

> 如果用户只发了 URL 没有明确指令，**先询问意图**，不要直接提取。

## 核心原则：用户核对优先

```
用户：发送 Confluence URL + "帮我提取这个文档内容"
    │
    ↓
AI 提取页面 → 输出 {页面标题}_整理版_v1.0.md
    │
    ↓
⏸️ 暂停！等待用户校验 MD
    │
    ├── 用户确认正确 → AI 标记"已确认"，等待下一步指令
    │
    ├── 用户指出错误 → AI 根据描述修正 MD → 再次等待校验
    │
    └── 用户补充说明 → AI 将补充内容写入 MD → 再次等待校验
    │
    ↓
用户说"评审这个需求" → 进入步骤② 需求评审
```

> 提取完成后不自动触发评审！必须等待用户明确指令。

## 执行步骤

### 步骤 0：确认用户故事信息并定位工作目录 ⭐

**执行逻辑**：

```
1. 向测试人员询问当前操作对应的 DMP 用户故事编号和用户故事名称
   - 格式：用户故事编号 + 用户故事名称
   - 示例：US-001 + 贷款审批流程优化
2. 构建工作目录路径：D:\效贷-产品需求\{用户故事编号}-{用户故事名称}
   - 示例：D:\效贷-产品需求\US-001-贷款审批流程优化
3. 检查该目录是否存在：
   - 不存在 → 自动创建该目录
   - 已存在 → 直接使用（说明"已有该用户故事目录，后续文件将存入该目录")
4. 将用户故事目录路径缓存到会话上下文（变量名：{user_story_dir}）
5. 后续所有步骤的输出文件统一存放到该用户故事目录下
```

**输出格式**：

```
📋 用户故事信息确认：

- 用户故事编号：US-001
- 用户故事名称：贷款审批流程优化
- 工作目录：D:\效贷-产品需求\US-001-贷款审批流程优化
- 目录状态：✅ 已创建 / ✅ 已存在

后续所有步骤的输出文件将存放到该目录。
```

> ⚠️ **如会话中已缓存用户故事目录路径**（测试人员之前已确认），直接使用缓存路径，跳过本步骤的询问环节，但仍需执行目录存在性检查。

---

### 步骤 1：提取 URL / pageId ⭐

从用户消息中提取 Confluence 页面地址。支持格式：

- `https://your-confluence-server/pages/viewpage.action?pageId=123456`
- `https://your-confluence-server/display/SPACE/Page+Title`
- `123456`（纯 pageId，需结合环境变量 CONFLUENCE_URL）

**如果用户没有提供 URL**：

```
请提供需要提取的 Confluence 页面链接，或告知 pageId。
```

---

### 步骤 2：确认输出目录（使用用户故事工作目录） ⭐

**直接使用步骤0确认的用户故事工作目录** `{user_story_dir}`。

> ⚠️ **不再询问用户单独指定输出目录**。输出目录统一为 `D:\效贷-产品需求\{用户故事编号}-{用户故事名称}`，由步骤0确认并缓存。

如果会话中未缓存 `{user_story_dir}`，需先返回步骤0确认用户故事信息。

---

### 步骤 3：通过 MCP 连接器提取页面内容 ⭐

**前提条件**：WorkBuddy 中已配置并启用 Confluence MCP 连接器。

配置路径：【专家.技能.连接器】-【连接器】-右上角【自定义连接器】-点击【配置MCP】，在 `mcp.json` 中添加 Confluence 连接器配置。

**提取流程**：

1. **预检**：确认 Confluence MCP 连接器已启用
   - 检查 `mcp__confluence__confluence_test_connection` 是否可用
   - 如果不可用，提示用户按上述路径配置 MCP 连接器

2. **读取页面内容**：
   - 如果用户提供的是 URL → 调用 `mcp__confluence__confluence_get_page_by_url`
   - 如果用户提供的是 pageId → 调用 `mcp__confluence__confluence_get_page`
   - 如果页面内容较长 → 调用 `mcp__confluence__confluence_get_page_body_chunk` 分段读取

3. **读取附件/图片**（可选）：
   - 调用 `mcp__confluence__confluence_get_page_attachments` 获取页面附件列表
   - 图片附件下载到 `source/images/`

4. **转换为 Markdown**：
   - 将 Confluence 存储格式（HTML/XHTML）转换为 Markdown
   - 保留页面原始标题、层级结构、表格、列表等格式
   - 图片引用改为本地路径 `source/images/{文件名}`

**⚠️ 安全约束**：
- 只读取，不修改 Confluence（仅调用 get 类工具，不调用 create/update/delete）
- 不得暴露内部服务器地址
- 敏感内容页面需先询问用户确认

---

## 与步骤②「需求评审」的衔接

当用户回复「确认整理完成」后，整理版 MD 已准备就绪。

**用户进入步骤②的方式**：

| 用户指令 | AI 行为 |
|---------|---------|
| `评审这个需求` | 读取当前 `{页面标题}_整理版_v1.0.md`，按 `requirement_review.md` 执行评审 |
| `评审这个目录` | 询问具体目录，进入 `document_consolidate.md` 或 `requirement_review.md` |
| `走完整流程` | 从步骤②开始继续执行（步骤①已完成） |

**需求评审前置检查**（requirement_review.md 0-3）看到 `*整理版*.md` 存在即可继续，不区分来源是本地目录还是 Confluence。

## 注意事项

1. **Confluence MCP 连接器**：如果 MCP 连接器未配置或未启用，提示用户按配置路径配置。配置路径：【专家.技能.连接器】-【连接器】-右上角【自定义连接器】-点击【配置MCP】。**不需要安装额外 skill**。
2. **页面权限**：如果页面 404/403，告知用户检查 URL 或联系 Confluence 管理员开通权限。
3. **大页面处理**：如果页面内容很长，提取时间可能较长，提前告知用户。
4. **多页面处理**：如果用户要求处理多个 Confluence 页面，先列出页面列表，经用户确认后再逐个执行。
5. **敏感内容**：页面标题包含"机密/保密/内部/confidential/internal"等关键词时，暂停执行并询问用户确认。
