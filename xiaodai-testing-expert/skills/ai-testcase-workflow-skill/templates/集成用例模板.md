# 集成测试用例模板

> 用于跨模块联调、端到端流程、数据同步、第三方集成类测试用例

---

## 用例结构

```markdown
### {用例编号}：{用例名称}

**用例类型**：集成测试
**用例级别**：P0（核心流程）/ P1（一般流程）
**功能路径**：{模块}-{跨模块流程名称}

**前置条件**：
- {系统1前置条件}
- {系统2前置条件}
- {接口配置}
- {数据预置}

**步骤**：
1、{步骤1：触发流程}
2、{步骤2：验证系统1状态}
3、{步骤3：验证系统2状态}
4、{步骤4：验证数据同步}

**预期结果**：
1、{结果1：流程触发成功}
2、{结果2：系统1状态正确}
3、{结果3：系统2状态正确}
4、{结果4：数据一致}

**来源**：技术方案/需求文档
**关联需求**：{需求编号}
**涉及接口**：{接口名称/编号}
```

---

## 集成测试用例设计要点

| 要点 | 说明 |
|------|------|
| **接口调用** | 每个新增接口至少覆盖：正常调用、异常返回、超时处理 |
| **数据同步** | 每个数据同步点至少覆盖：同步成功、同步失败、数据一致性 |
| **流程联调** | 每个跨模块流程至少覆盖：完整流程、中断流程、异常分支 |
| **状态联动** | 每个状态联动至少覆盖：联动成功、联动失败 |
| **第三方集成** | 每个第三方依赖至少覆盖：正常响应、异常响应、超时处理 |

---

## 常见集成测试场景

| 场景类型 | 测试要点 |
|---------|---------|
| **新增接口调用** | 参数传递正确、返回值处理、异常处理、超时处理 |
| **接口参数变更** | 新参数传递正确、兼容老参数 |
| **数据同步新增** | 同步时机正确、数据完整、一致性验证 |
| **流程新增** | 流程完整、状态流转正确、异常处理 |
| **第三方集成** | 调用正确、返回处理、降级方案 |

---

## 集成用例占比要求

- 集成测试用例数占（功能+集成）总用例数的 **20%~40%**
- 跨模块的功能场景均归入集成测试
- 核心流程的集成测试用例级别为 P0
